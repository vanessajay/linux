Implement and Manage Disk Quotas in Linux Implementar e gerenciar cotas de disco no Linux

Posted by Admin on September 22nd, 2006 Postado por Admin em 22 de setembro de 2006

If you're new here, you may want to subscribe to my RSS feed . Se você é novo aqui, você pode querer assinar o meu feed RSS . Thanks for visiting! Obrigado pela visita!

A disk quota is a limit set by a system administrator that restricts certain aspects of file system usage on modern operating systems.Disk quotas are very useful in large multi-user environments where you have a large number of users with access to the filesystem, whether because they log into the system or use their file space remotely, through either Samba or NFS.This is very useful if you are an Internet service provider (ISP) providing web space for your users to host their websites. A cota de disco é um limite definido pelo administrador do sistema que restringe a determinados aspectos da utilização do sistema de arquivos em funcionamento quotas systems.Disk modernos são muito úteis em ambientes multi-usuário grande onde você tem um grande número de usuários com acesso ao sistema de arquivos, se porque se logar no sistema ou utilizar o seu espaço de arquivos remotamente, quer através de Samba ou NFS.This é muito útil se você for um prestador de serviços de Internet (ISP), oferecendo espaço web para seus usuários para hospedar seus sites.

Debian supports BSD-style disk quotas on the ext2 filesystem. Debian suporta BSD quotas de disco no sistema de arquivos ext2. This allows you to limit the amount of disk space that your users consume. Isto permite-lhe limitar a quantidade de espaço em disco que os usuários consomem. QUOTA is implemented using the BSD system call interface as the means of communication with the user level. QUOTA é implementado usando o BSD interface chamada de sistema como meio de comunicação com o nível de usuário. This is based on the Melbourne quota system which uses both user and group quota files. Esta baseia-se no sistema de quotas de Melbourne, que utiliza tanto o usuário e os arquivos de quota do grupo.

Installing Quota in debian Instalando Quota no Debian

#apt-get install quota quotatool # Apt-get install quota quotatool

At the time of installation time it will prompt the following questions No momento do tempo de instalação irá pedir as seguintes perguntas

send daily remainders to users over quota with the following screen here select yes and press enter enviar restos diários para os usuários, com quota com a seguinte tela aqui selecione Sim e pressione Enter

next screen you need to enter the support email of the admin select ok and press enter próxima tela, você precisará digitar o e-mail de apoio do administrador ok selecionar e pressione enter

Here you need to enter the phone support number of the admin select ok and press enter. Aqui você precisará digitar o número de telefone de apoio ao administrador ok selecionar e pressione enter.

This will complete the quota installation. Isto irá completar a instalação do contingente.

Configuring Disk Quotas Configurando cotas de disco

Edit /etc/fstab to look like this (I added ,usrquota,grpquota to the partition with the mount point /): Edite o arquivo / etc / fstab para ficar assim (eu acrescentei, usrquota, grpquota à partição com o ponto de montagem /):

# /etc/fstab: static file system information. # / Etc / fstab: informações estáticas do sistema de arquivos.

# #

# #

proc /proc proc defaults 0 0 proc / proc proc defaults 0 0

/dev/sda1 / ext3 defaults,errors=remount-ro,usrquota,grpquota 0 1 / Dev/sda1 / ext3 defaults, errors = remount-ro, usrquota, grpquota 0 1

/dev/sda5 none swap sw 0 0 / Dev/sda5 none swap sw 0 0

/dev/hdc /media/cdrom0 iso9660 ro,user,noauto 0 0 / Dev / hdc / media/cdrom0 iso9660 ro, user, noauto 0 0

/dev/fd0 /media/floppy0 auto rw,user,noauto 0 0 / Dev/fd0 / media/floppy0 auto rw, user, noauto 0 0

Now you need to Enable quota for this you need to run the following commands Agora você precisa ativar quota para isso você precisa executar os seguintes comandos

#touch /quota.user /quota.group # Touch quota.user / quota.group

#chmod 600 /quota.* # Chmod 600 / quota .*

#mount -o remount / # Mount-o remount /

When each quota-enabled file system is remounted, the system is now capable of working with disk quotas. Quando cada quota-enabled sistema de arquivos é remontado, o sistema agora é capaz de trabalhar com quotas de disco. However, the No entanto, o
file system itself is not yet ready to support quotas. próprio sistema de arquivo ainda não está pronto para suportar quotas. To do this, you must first run quotacheck. Para fazer isso, você deve primeiro executar quotacheck.

The quotacheck command examines quota-enabled file systems, building a table of the current disk usage for each one. O comando quotacheck examina contingente habilitado para sistemas de arquivos, a construção de uma tabela do uso corrente do disco para cada um.
This table is then used to update the operating system's copy of disk usage. Esta tabela é usada para atualizar a cópia do sistema operacional de uso do disco. In addition, the file system's disk quota files are updated (or created, if they do not already exist). Além disso, os arquivos do sistema de arquivos de cota de disco é atualizada (ou criadas, se não existir).

#quotacheck -avugm # Quotacheck avugm-

Check all quota-enabled, locally-mounted file systems (-a) Confira todas as quotas habilitado montados localmente com sistemas de arquivos (-a)

Display status information as the quota check proceeds (-v) Exibir informações de status como o produto contingente de seleção (-v)

Check user disk quota information (-u) Confira as informações do usuário de quota de disco (-u)

Check group disk quota information (-g) Confira as informações de cota de disco do grupo (-g)

Don't try to remount filesystem read-only (-m) Não tente montar o sistema de arquivos read-only (m)

Now you need to run quotaon command.quotaon announces to the system that disk quotas should be enabled on one or more filesystems. Agora você precisa executar command.quotaon quotaon anuncia para o sistema de cotas de disco que deve ser ativado em um ou mais arquivos.

#quotaon –avug # Quotaon-avug

Force all filesystems in /etc/fstab to have their quotas disabled (-a) Força todos os arquivos em / etc / fstab para ter as suas quotas com deficiência (-a)

Display a message for each filesystem affected (-v) Mostrar uma mensagem para cada sistema de arquivos afetados (-v)

Manipulate user quotas (-u) Manipular as quotas de usuários (-u)

Manipulate group quotas (-g) Manipular quotas grupo (-g)

If you want to manage quotas with graphical webinterface you need to install quota webmin module. Se você deseja gerenciar quotas com webinterface gráfica você precisa instalar o módulo webmin quota.

Before installing webmin module make sure that you have installed webmin in your server.If you don't have webmin installed in your server follow this article to install . Antes de instalar o módulo webmin certifique-se que você tem instalados no seu webmin server.If webmin você não tenha instalado em seu servidor siga este artigo para instalar .

To Install webmin module for quotas run the following command Para instalar o módulo webmin quotas para executar o seguinte comando

#apt-get install webmin-quota # Apt-get install webmin quota-

Now you login in to the webmin and you can see quota configuration Agora, você entra para o webmin e você pode ver a configuração de quotas

check Warnquota is configured or not verificar Warnquota está configurado ou não

- email warnings are triggered by warnquota - Advertências e-mail são acionados por warnquota
- warnquota takes its configuration in /etc/warnquota.conf - Warnquota tem sua configuração em / etc / warnquota.conf
- warnquota is ran daily by default : /etc/cron.daily/quota - Warnquota está correndo diariamente por padrão: / etc / cron.daily / quota

Here is my /etc/warnquota.conf Aqui está o meu / etc / warnquota.conf

MAIL_CMD = “/usr/sbin/sendmail -t” MAIL_CMD = "/ usr / sbin / sendmail-t"
FROM = “your.address@your.domain.com” FROM = "your.address your.domain.com @"
SUBJECT = You have exceeded your quota Subject = Você excedeu sua cota
CC_TO = “your.address@your.domain.com” CC_TO = "your.address your.domain.com @"
SUPPORT = “support.address@your.domain.com” APOIO = "support.address your.domain.com @"
PHONE = “01 02 03 04 05″ TELEFONE = "01 02 03 04 05"
MESSAGE = “quota exceed for this user” MENSAGEM = "quota superior para este usuário"
SIGNATURE = “Your signature” ASSINATURA = "Sua assinatura"

Try dpkg-reconfigure -plow quota, it should ask you questions about warnquota configuration. Tente dpkg-reconfigure-plow quota, deve fazer-lhe perguntas sobre a configuração warnquota.

BTW, if your use of quota is critical, test it before production or with a test account by lowering quota to a very small level and by running manually /etc/cron.daily/quota. BTW, se o uso de quotas é fundamental, testá-lo antes da produção ou com uma conta de teste através de uma redução do contingente a um nível muito pequeno e executando manualmente o arquivo / etc / cron.daily / quota.
